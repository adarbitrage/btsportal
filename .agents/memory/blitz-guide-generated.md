---
name: Blitz guide is generated HTML
description: The Blitz guide page body is generated by a build script and injected as a raw HTML string. Critical constraint for any restyle of the guide.
---

**Rule:** The Blitz Guide page (`artifacts/portal/src/pages/Blitz.tsx`) renders its body as one large raw HTML string injected via `dangerouslySetInnerHTML`, scoped/styled by the `blitzCSS` constant. That HTML is **generated** by `artifacts/api-server/src/scripts/build-blitz-from-html.ts` from a source HTML file in `attached_assets/`.

**Why it matters:** Hand-editing the HTML structure or its class names inside `Blitz.tsx` will be **overwritten** the next time the guide is regenerated from source. The runtime relies on specific hooks in that HTML: `data-section` attributes (deep-link filtering), `id="sN"` anchor spans, `.module` / `.mod-badge` / `.path-tag` classes, and `MODULE1_OVERRIDES` / `LESSON_LOOKUP` logic.

**How to apply (restyle the guide safely):**
- Prefer restyling via the scoped `blitzCSS` block (and the page frame around the injected HTML) rather than rewriting the HTML markup, so regeneration stays compatible.
- If markup/class changes are unavoidable, change the **generator** (`build-blitz-from-html.ts`) and the source HTML, not just the rendered output — otherwise the next build reverts them.
- Functionality to preserve in any restyle: deep-linking via `:lessonId`, per-path visibility overrides (Media Mavens vs ClickBank), the Vidalytics video lightbox, and the bottom Lesson Library.

**Proven restyle technique (used for the portal-token restyle):** retarget the `.blitz-content` root CSS custom properties (`--primary/--bg/--card/--border/--text/--muted`) to the site's `hsl(...)` token values + swap `font-family` to Roboto, then **append override rules** at the end of `blitzCSS` to convert the few dark/navy surfaces (`.page-header`, `nav.toc`, `th`, `.section-nav-bar`, `.version-banner`) to light treatment. `SECTION_BAR_CSS` is a separate const that must be restyled too. This is far smaller/safer than rewriting all ~375 CSS lines, and propagates via the cascade. **Any such guide-CSS restyle must be ported into `build-blitz-from-html.ts` before the next regen, or it is lost.**
