---
name: Blitz section-doc rebuild (replaced old reference import)
description: The Blitz KB corpus is now 29 section-anchored generated docs; the old 90+ lesson-shaped corpus and its blanket importer are retired. How the rebuild pipeline, idempotency, and drift guards work.
---

The original lesson-shaped Blitz corpus (~94 `reference_docs` sources mirrored
from `blitz_lessons`, imported via `POST .../import-blitz-references`) was
HARD-DELETED (July 2026) and replaced by 29 section-anchored docs generated by
gpt-5 from BLITZ_BODY_HTML section text (authoritative) + the 44 `blitz_video`
transcripts (enrichment only).

**Pipeline (api-server):** `lib/blitz-section-extract.ts` (splits guide HTML
into 23 sections via the shared badge tokenizer; attributes transcripts by
parsing "Lesson N · section · video" titles — hard-fails on any drift) →
`lib/blitz-section-docgen.ts` (fixed 29-doc manifest; sections 7/8/9/19 split
by exact video-title pins in exported `SECTION_SPLITS`; 8500-char cap, one
condense retry, loud fail) → runner `scripts/blitz-section-rebuild-run.ts`
(resumable, per-doc commit).

**Idempotency:** source rows keyed on exact title + `sourceType='reference_docs'`;
staging rows on (source=`blitz_section_import`, sourceVideoTitle=exact title).
Renaming a manifest title creates a duplicate on re-run.

**Drift guard:** `blitz-section-manifest-drift.test.ts` pins the 29-title set;
its synthetic transcript corpus is built FROM `SECTION_SPLITS` pins because
real transcript title tails do NOT match the video-map slot titles (they are
Title-Cased file names from the Transcript Cleaner import).

**Retired seams (do not resurrect):** `importBlitzReferenceDocs`, the
`/import-blitz-references` route, the portal import button, and the
`blitz_lessons` mirror in `seed-core-training-sources.ts` (seed now files ONLY
the 2 prose docs). `BLITZ_REFERENCE_IMPORT_SOURCE` + transforms remain for
legacy-row scoping in kb-triage; 3 legacy staging keepers + 2 prose sources
survive. `kb_node_synthesis_state.sourceDocIds` was left stale on purpose —
it marks nodes needing re-synthesis.

**Approval-gate quirks (still apply to the new docs):**
- `single_source` fires on every curriculum doc — expected, dismiss via flags/resolve.
- `push-approved` pushes ALL approved docs corpus-wide.
