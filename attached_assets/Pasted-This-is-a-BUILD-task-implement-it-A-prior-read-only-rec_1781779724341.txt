## API ENDPOINTS
- POST /api/voice/web-call — auth required (global authenticate). Gate inline with hasEntitlement(req.userId, "voice:access") + the explicit admin-bypass pattern from community.ts:35–47. Then enforce cap: read today's seconds_used from voice_daily_usage; if >= VOICE_DAILY_SECONDS_CAP return 403 { error, code: "voice_cap_reached" }. Else build ctx = buildMemberVoiceContext(userId) and call the Retell server SDK createWebCall with agent_id = process.env.RETELL_AGENT_ID, retell_llm_dynamic_variables = ctx, metadata = { bts_user_id: String(userId) }. Insert a voice_calls row (retell_call_id, status "registered", started_at now). Return { access_token, call_id }. Fail closed (500 + clear log) if RETELL_API_KEY or RETELL_AGENT_ID is unset. Confirm the exact SDK method path (e.g. client.call.createWebCall) against the installed retell-sdk types.
- GET /api/voice/status — auth required. Return { has_access, daily_cap_seconds, seconds_used_today, seconds_remaining }.
- POST /api/voice/kb-search — PUBLIC (no member auth; server-to-server from Retell). Require header Authorization: Bearer <RETELL_FUNCTION_SECRET>; in production reject 401 (timingSafeEqual) if the secret is unset or mismatched — same fail-closed posture as the webhook receivers. Body { query: string }. Run searchKnowledgebase(query, <FULL category set>) — comprehensive, NOT tier-scoped (deliberate: the feature is gated at access, the knowledge is intentionally full for all members). Return { results: string } as concise plain text suitable to be read aloud; cap the length for voice.
- POST /api/webhooks/retell — PUBLIC webhook. Verify signature against req.rawBody using RETELL_API_KEY (the signing secret) and the Retell signature header; fail closed exactly like webhooks-ghl.ts (prod rejects invalid/missing). Confirm the verify method + exact header name against current Retell docs (Secure Webhook section). Handle: call_started (update status); call_ended (set ended_at, duration_seconds, status, disconnect_reason; UPSERT today's voice_daily_usage += duration_seconds); call_analyzed (set summary/transcript/analysis). Return 200 after a successful verify+handle; early-200 for unknown event types.
