**Implementation details:**

1. The top-level "Admin" item is a collapsible folder with a chevron icon. Clicking it expands/collapses the entire admin section. Persist the expanded/collapsed state in localStorage so it remembers between page loads.

2. Sub-folders (Content, Support, Coaching, etc.) are also collapsible. Clicking a sub-folder expands just that section. When you navigate to a page within a sub-folder, that sub-folder auto-expands.

3. The active page should be highlighted AND its parent folder(s) should be auto-expanded.

4. Admin nav items are ONLY rendered if the user has an admin role (admin, super_admin, support_agent, content_manager). Role-based filtering: support agents only see the Support sub-folder and Knowledge Base. Content managers only see Content and Knowledge Base.

5. On mobile: the admin section should be at the bottom of the navigation, clearly separated from member-facing nav with a divider and "Admin" label.

6. Visual distinction: admin nav items should have a slightly different style (subtle background tint or left border) so it's visually clear these are admin functions, not member features.

7. Move ALL current admin pages under `/admin/*` routes if they aren't already. The sidebar should map to these routes.

8. Keep "Back to Portal" link at the bottom of the admin section so admins can quickly switch back to the member view.

This will clean up the nav significantly. Right now everything's flat — once it's foldered, a member sees ~10 clean nav items and an admin sees the same 10 plus one expandable "Admin" folder.
