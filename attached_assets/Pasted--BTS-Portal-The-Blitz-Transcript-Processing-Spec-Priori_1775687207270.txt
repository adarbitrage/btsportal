## Context

The Blitz is BTS's flagship A-to-Z training program — 81 videos covering the complete Build, Test, Scale framework for affiliate arbitrage. Unlike the previous 97 coaching transcripts (unstructured, overlapping topics), The Blitz is a structured, sequential curriculum with clear phases, modules, and branching paths based on affiliate network and publisher.

**Critical differences from the coaching transcript batch:**
- Each video = one standalone training document (no deduplication/merging)
- Sequence order matters — Lesson 1 before Lesson 2 before Lesson 3
- Module/phase tagging is essential for the training system
- Branching paths must be preserved (Media Mavens vs ClickBank vs MaxWeb, Caterpillar vs Grasshopper vs Crane)
- The page text below provides the FULL curriculum structure, lesson order, and context for each video

## Processing Instructions for Replit

### Step 1: Fetch and Match Videos

The 81 Blitz videos are in a single Vidalytics folder, unordered. You need to match each video to its correct position in the curriculum above.

```
1. Fetch all videos from the Blitz folder via Vidalytics API (title + MP4 URL)
2. For each video, match its title to a lesson in the curriculum map above
3. Use fuzzy matching — video titles in Vidalytics may not match exactly:
   - "Clone Flexy Website" might be "How to Clone a Flexy Website" in Vidalytics
   - "MetricMover 3" might be "How To Import Your Landing Page Into MetricMover"
   - Match on key phrases, not exact strings
4. Output a mapping: { vidalyticsTitle, vidalyticsUrl, lessonId, moduleName, phase }
5. Flag any unmatched videos for manual review
6. Flag any curriculum lessons with no matching video
```

### Step 2: Transcribe

Same as the previous batch:
```
1. Download each video, extract audio with ffmpeg
2. Transcribe via OpenAI Whisper
3. For long videos (40+ min): split into chunks for transcription
4. Store raw transcripts with lesson mapping metadata
```

### Step 4: Extract Training Documents

Use the same Claude extraction pipeline as before, but with a **Blitz-specific prompt** that preserves curriculum structure:

```
SYSTEM PROMPT (Blitz Extraction):

You are converting a training video transcript into a clean, structured 
training document. This video is part of "The Blitz" — a sequential, 
step-by-step affiliate marketing training program.

CRITICAL CONTEXT:
- Phase: {{phase}} (BUILD, TEST, or SCALE)
- Module: {{module_name}}
- Lesson: {{lesson_id}} — "{{lesson_title}}"
- Lesson Type: {{lesson_type}} (conceptual, technical-walkthrough, or strategy)
- Network Path: {{network_path}} (universal, media-mavens, clickbank, maxweb, or publisher-specific)

This lesson is part of a specific sequence. The member completed previous 
lessons before reaching this one. Write the document assuming the reader 
has that prior context.

RULES:
- Write in clear, direct prose. No verbal fillers or conversational padding.
- Organize with clear headings (## and ###)
- For technical walkthroughs: extract the exact step-by-step process as 
  numbered lists. Be precise about which buttons to click, which fields 
  to fill in, and what the expected result is.
- For conceptual training: extract the key frameworks, principles, and 
  mental models being taught. Include specific examples.
- For strategy content: extract the decision criteria, success metrics, 
  budgets, and rules of thumb.
- Preserve ALL specific numbers: budgets ($1,500, $500, $25/banner), 
  metrics (20% ROI, 60% return threshold), character limits (90 chars), 
  image dimensions (960x540, 16x9), and timeframes.
- Preserve tool names exactly: Flexy, MetricMover, DIYTrax, PixelPress, 
  FreeAdCopy, AffiliateCMO, CropBot, Gifster, ScrapeBot
- Preserve publisher codenames: Caterpillar, Grasshopper, Crane, Master
- Preserve affiliate network names: Media Mavens, ClickBank, MaxWeb
- If this lesson applies to a specific network or publisher path, note 
  that clearly at the top: "This lesson applies to: [MEDIA MAVENS] members"
- Remove all verbal filler (uh, um, you know, basically, etc.)
- Remove tangents and off-topic asides
- Do NOT add information not in the transcript
- BTS branding only. Never use TCE, Cherrington, or Adam's name.
- Target length: 300-1000 words depending on lesson complexity.
  A 5-minute technical walkthrough might be 300 words of clear steps.
  A 30-minute conceptual lesson might be 800-1000 words.

OUTPUT FORMAT:

# {{lesson_id}}: {{lesson_title}}

**Phase:** {{phase}}
**Module:** {{module_name}}
**Category:** {{category}}
**Applies to:** {{network_path}}
**Topics:** {{comma-separated tags}}

## Overview

[1-2 sentence summary of what this lesson teaches and why it matters 
at this point in the Blitz]

## [Main Content — Headings appropriate to the lesson type]

[Clean, structured content...]

## Key Takeaways

- [Bullet point summary of the most important points]

## Next Step

[What the member should do after completing this lesson — 
reference the next lesson in the sequence if applicable]
```

### Step 6: Tag Everything Correctly

Every document gets metadata tags for the knowledge base:

```json
{
  "source": "blitz",
  "phase": "build",           // build, test, scale
  "module": "offer-selection", // module slug
  "lesson_id": "2.3",         // lesson number
  "lesson_type": "technical",  // conceptual, technical, strategy
  "network_path": "media-mavens", // universal, media-mavens, clickbank, maxweb
  "publisher_path": "all",     // all, caterpillar, grasshopper, crane
  "tools_mentioned": ["media-mavens", "diytrax"],
  "blitz_order": 7            // sequential position (1-81+)
}
```

This metadata is critical for the AI chat — when a member asks "how do I set up my ClickBank campaign in DIYTrax?", the RAG system needs to pull the ClickBank-specific lessons, not the Media Mavens ones.

### Step 7: Stage and Review

All documents go to `kb_staging_docs` with status `pending_review`, tagged with the Blitz metadata. Admin reviews in the same UI as before.

**Review priority:** Since this is the core training, Adam should review these more carefully than the coaching transcripts. Suggested review order:
1. Conceptual lessons (frameworks, strategies) — these shape understanding
2. Page-text documents (budgets, criteria, publisher details) — these contain exact numbers
3. Technical walkthroughs (click-by-click steps) — these are most likely to be accurately extracted
