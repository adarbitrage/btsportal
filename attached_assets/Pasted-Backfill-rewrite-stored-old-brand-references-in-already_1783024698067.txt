Backfill: rewrite stored old-brand references in already-cleaned AI source content

Task #1604 added old-brand → BTS rebrand rules to the shared privacy filter (artifacts/api-server/src/lib/content-privacy-filter.ts — OLD_BRAND_REBRAND_GUIDANCE plus the old-brand block inside PRIVACY_RULES). Those apply during new transcript cleans/refine and at retrieval time, but they do not rewrite content that was already cleaned into the Transcript Cleaner holding store or already filed into the AI Source Knowledge library. Those rows still carry the old brand in their stored text; the retrieval-time filter neutralizes it on the way out, but the stored source-of-truth is off-brand and downstream mining/synthesis reads that stored text.

Important — don't duplicate existing coverage. Seeded knowledgebase_docs rows are already handled: the existing rescrub-knowledgebase-docs pass (run on boot in bootstrap-critical-prerequisites.ts and again in scripts/post-merge.sh) applies the full scrubPrivateContent, which now includes the #1604 old-brand rules. The real gaps are the two tables that have no rescrub pass:

transcript_cleaner_documents (holding store): cleaned_content, and the title fields title / suggested_title / proposed_title.
ai_source_documents (filed raw-source library): content, title.
Scope discipline — old-brand ONLY. Do NOT run the full privacy filter over these two tables. They are raw mining input where coach-name attribution is meaningful (authority labelling), so stripping coach surnames here would damage the data and is out of scope. Rewrite only the old-brand references, reusing the exact rule objects from the shared filter so it stays in lockstep with the cleaner.

Implement (follow the rescrub-knowledgebase-docs precedent):

In content-privacy-filter.ts, factor the old-brand rules out of PRIVACY_RULES into an exported OLD_BRAND_REBRAND_RULES: PrivacyRule[] (the "Adam Cherrington → Adam" rule and the company/program → BTS rules, plus the trailing whitespace-collapse cleanup rules needed after them). Keep PRIVACY_RULES composed from that same array so there's one source of truth. Add and export rebrandOldBrandContent(text): string that applies only OLD_BRAND_REBRAND_RULES, with the same null/empty handling as scrubPrivateContent.
