Blitz Reference Docs — Lesson Provenance & Publish Readiness

BTS Member Portal · AI Document Review · July 16, 2026
Verdict: No data was lost during the import of the 96 Blitz reference docs, no new import rules are needed, and the publish path to the live AI knowledge base already carries everything the AI assistant needs to direct members to the correct Blitz sections.

1. What the question was

When the 96 reference docs from AI Source Knowledge were imported into the AI Document Review section, their display names lost the internal numerical system (e.g. “4.4”, “3.18b”) that identified which Blitz lesson and part each doc came from. Was that provenance lost, and can the AI assistant still send members to the right place in the Blitz?

2. Where the lesson provenance is stored

Every one of the 96 imported docs retains its origin in three places:

FieldWhat it holdsExample (“Submit Ad Split Test Media to Compliance”)
source_video_titleThe exact original AI Source Knowledge title, numbering included. Also the import’s idempotency key — all 96 match their source rows exactly.The Blitz™ Lesson — 4.4: Submit Ad Split Test Media to Compliance
lesson_idThe parsed internal lesson number, queryable on its own.4.4
blitz_sectionThe member-facing Blitz guide section the lesson belongs to, resolved from a drift-guarded crosswalk against the real curriculum.10

81 of the 96 docs carry a lesson_id; the other 15 never had a number in their original titles (guides and overviews such as “Your Blitz Roadmap”, “Round 1 Campaign Management Guide”, and the two Core Training docs). That is not data loss — there was no number to carry over.

3. Why numbering was removed from titles and bodies — by design

The internal numbering is mining scaffolding that members never see anywhere in the portal. The live Blitz guide is organized by phase and section, not by those numbers, so a member told to “go to Lesson 3.18b” would have nothing to find. The importer therefore translates rather than deletes: every cross-reference such as “Proceed to Lesson 4.5” was rewritten to the canonical member-facing anchor:

Section 10 (“Submit Your Assets for Compliance Review”) in the Build phase of the Blitz guide, where you will learn how to integrate your ad image into your campaign setup…

The assistant’s system prompt enforces the same convention: refer to the Blitz textually in exactly this form, and never repeat internal lesson numbers.

4. Accuracy audit of the rewritten references

All 38 rewritten anchor phrases across the 96 docs were audited against the real curriculum structure (sections 1–2 Introduction, 3–14 Build, 15–20 Test, 21–23 Scale). Result: zero mismatches. The phrases cannot easily drift, because the section number and phase name are generated from the same curriculum skeleton that drives the live Blitz guide.

5. Publish path to the live AI knowledge base

The approve-and-push flow was verified end to end in code. Everything that matters survives publishing:

  • Rewritten cross-references live inside the document body, which is copied verbatim at push time. The only transform applied is the privacy scrub (emails, phone numbers, coach surnames), which cannot affect the anchor phrases.
  • blitz_section is explicitly carried onto the live AI document in all three write paths (in-place update, new insert, and title-conflict upsert), with an existing test guarding section→taxonomy consistency.
  • Provenance rows are written to a dedicated table on every push, tracing each live doc back to its screened source — the trail to the original numbered lesson is preserved even in the live knowledge base.

6. What remains (process, not code)

  • The 96 docs go through the normal review → approve → push flow in AI Document Review (the human gate).
  • The review pipeline runs in dev only; approved outputs are promoted to production in a one-time step that carries the same rows (content, blitz_section, provenance) unchanged.

Once reviewed and approved, the AI assistant will direct members to the correct Blitz section, with correct phase wording, in the same language the Blitz guide itself uses.
